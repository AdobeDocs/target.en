# Access Flags in Adobe Target {#log-in}

To get started with Flags, request access and then open Flags from within Adobe Target.

## Request access {#request-access}

Request access through your organization's access management system. See [Request access](request-access.md) for step-by-step instructions.

After your request is approved, you will receive the permissions associated with the role granted by your admin.

## Access Flags {#log-in-steps}

After access is granted:

1. Go to [https://experience.adobe.com/](https://experience.adobe.com/) and sign in with your organization credentials.
1. Open **Adobe Target**.
1. Select **Flags** from the Target navigation.
1. Select the correct sandbox. See [Environments overview](environments-overview.md) for details.

## First steps after accessing Flags {#first-steps}

After opening Flags, verify that your application is listed. Applications are managed by admins. If your application is not listed, contact your admin to have it added. See [Manage applications](../applications/manage-applications.md) for more information.

## Key terminology {#terminology}

| Term | Definition |
| --- | --- |
| **Application** | The application you want to control with feature flags. |
| **Feature flag / Feature group** | The artifacts created in Flags for feature testing and controlled rollouts. |
