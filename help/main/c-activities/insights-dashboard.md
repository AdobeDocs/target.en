# Adobe Target Insights Dashboard

The [!UICONTROL Adobe Target Dashboard] provides a high-level view of how your organization is using Target over time. It helps teams understand adoption, activity volume, and experimentation usage at a glance.

This page is designed for both practitioners and stakeholders who want quick visibility into Target usage without needing to analyze individual reports.

## Header Callout: Experimentation Accelerator

At the top of the page, the dashboard includes a brief header message that references [!UICONTROL **Experimentation Accelerator**]. This callout is intended to make customers aware of additional capabilities that help streamline experimentation workflows. The message serves as a lightweight entry point for customers who want to explore tools designed to simplify experiment setup, analysis, and decision-making.

## Time Range Selection

At the top of the page, you can select a time range (for example, the last week, last year or all time).

The selected time range applies consistently across all metrics and charts on the page.

**Important to know:**

* Some metrics reflect activities that were live at any point during the time range.
* Others reflect activities that were created, published, or completed within the time range.
* As a result, totals across metrics may not always add up exactly. For example, many activities could have been started and completed in the same time frame. 

## Summary Metrics

The summary metrics at the top of the page provide a quick snapshot of activity within the selected time range.

### Total Live Activities

This metric shows the number of activities that were live at any point during the selected time range.

An activity is considered live if it was actively serving traffic, even if it started before or ended after the selected period.

**Use this metric to:**

* Understand how actively Target was used during the time period
* Gauge the overall scale of personalization and testing efforts

### Activities Live or Modified

The total number of activities in your organization that have been live, created, or modified in the selected time frame.

**Use this metric to:**

* Understand the overall size of your Target activity library and how many activities are in use
* Track long-term growth of experimentation and personalization programs

### Completed Activities

This metric shows the number of activities that reached a completion or stop date during the selected time range.

**Use this metric to:**

* Understand how many activities concluded during the time period
* Track completion volume over time

### Activities Published

This metric shows the number of activities that were published during the selected time range.

An activity is considered published when it is made live for the first time. If an activity was made live, stopped, and then made live again, only the first occurrence will be counted in this metric. 

**Use this metric to:**

* Measure how many new activities were launched
* Understand activity creation and publishing velocity

## Activity Type Breakdown

The Activity Type chart shows the distribution of live activities by type, such as:

* A/B Test
* Experience Targeting (XT)
* Automated Personalization (AP)
* Multivariate Test (MVT)

The chart reflects total activities during the selected time range.

**Use this section to:**

* Understand which Target capabilities are most commonly used

## A/B Testing Metrics

This section highlights usage specifically related to A/B testing.

### Total Live A/B Test Activities

Shows the number of A/B test activities that were live at any point during the selected time range.

### Total A/B Tests Published

Shows the number of A/B test activities that were published during the selected time range.

**Use these metrics to:**

* Understand how frequently A/B testing is being used
* Track experimentation volume and adoption

## Interpreting the Data

When reviewing this dashboard, keep the following in mind:

* Metrics may include activities that started before or ended after the selected time range.
* An activity may be counted in multiple metrics depending on its lifecycle (for example, published and completed).
* This dashboard focuses on usage and adoption, not performance outcomes.
 
For detailed results, lift, or statistical performance, refer to individual activity reports within Adobe Target.

## Intended Use

This dashboard is ideal for monitoring Target adoption and usage trends. It is not intended to replace detailed activity-level analysis.
